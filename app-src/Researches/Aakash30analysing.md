## 1. Button Placement

* Every page should have **one main action button**
* Example: **Add Project**
* Best positions:

  * Top‚Äëright corner
  * Near the page title
* Avoid placing important buttons at the bottom

**Rule:** User should find the main button within **2 seconds**.

---

## 2. Page Layout Structure

* Divide the page into clear sections:

  * Header (title, buttons)
  * Content (cards, lists)
  * Footer (optional)
* Maintain space between sections
* Avoid overcrowding content

**White space = clean UI**

---

## 3. Cards Design

Cards are containers used to display information.

Each card should include:

* Title (bold)
* Short description
* One clear action (View / Edit)

**Good UI:** Clean, readable cards
**Bad UI:** Too much text, no spacing

---

## 4. Color Usage

* Use **one primary color** (blue/green)
* Use neutral colors for rest:

  * Black
  * White
  * Gray
* Avoid using many colors

**Example:**

* Buttons ‚Üí Blue
* Text ‚Üí White / Gray
* Background ‚Üí Dark gray / Black

---

## 5. Typography (Text Sizes)

* Page title ‚Üí Large
* Section titles ‚Üí Medium
* Body text ‚Üí Small

If everything is the same size, UI feels boring and confusing.

---

## 6. Hover Effects

Hover effects show that something is clickable.

Examples:

* Button color changes on hover
* Card slightly highlights

This improves user confidence.

---

## 8. Empty State Design

When no data is available:

* Do not show a blank page
* Show a message like:

  * "No projects yet"
* Provide a button:

  * "Add your first project"

This makes the app feel friendly.

---

## 9. Basic Accessibility Rules

* Buttons should be easy to click
* Text should be readable
* Avoid light text on light backgrounds

Accessibility improves usability for everyone.
